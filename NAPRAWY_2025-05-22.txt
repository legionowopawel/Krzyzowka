PROBLEM 2: Algorytm — Wyrazy nakładają się (KRYTYCZNE)
───────────────────────────────────────────────────────────────────────────

Przyczyna:
  - can_place_word() nie walidowała prawidłowo pozycji wzajemnie
  - _backtrack() pozwalał umieszczać wyrazy bez wymuszenia ich przecinania
  - Wynik: Wyrazy wpisywane na siebie, chaos (pppspsycholog)

Wdrażane zmiany:

  A) crossword_grid.py — can_place_word():
  
     Zmienione reguły walidacji:
     
     Przed:
       if cell and cell != letter:  # Zbyt liberalne
           return False
     
     Po:
       if cell is None:  # Odrzuć czarne komórki
           return False
       if cell != "" and cell != letter:  # Komórka zarezerwowana
           return False
     
     Efekt: Nie dopuszcza umieszczania liter na czarnych komórkach
             Wymaga dokładnego dopasowania liter w przecięciach

  B) crossword_generator.py — _backtrack():
  
     Dodana logika wymuszenia przecinania:
     
     - Nowa metoda: _requires_intersection()
       Zwraca True gdy filled_count > 10
       (Po umieszczeniu 10+ komórek, nowe wyrazy MUSZĄ się przecinać)
     
     - Zmodyfikowana pętla umieszczania:
       
       if self._requires_intersection(grid) and not intersection_count:
           continue  # Pomijaj wyrazy bez przecięcia
     
     - Zmniejszona liczba kandydatów:
       Przed: top 3 wyrazy na pozycję
       Po:   top 2 wyrazy na pozycję (bardziej restrykcyjne)

  C) Logika punktacji przecinań:
  
     _count_intersections() teraz dokładnie liczą pokrywające się litery
     Wyrazy z zerowym przecięciem są odrzucane gdy wymuszono przenikanie

Rezultat zmian w testach:
  Przed:
    Siatka: (chaos z nakładającymi się literami)
    Pytania: 0 (clue numbering broken)
  
  Po:
    Siatka: Prawidłowa struktura 10×10
    Słów: 15 (harmonijnie rozmieszczone)
    Pytania: Prawidłowo numerowane

Status: ✅ NAPRAWIONE
Test: ✓ CLI: python main.py --cli 10 10 baza.txt
Verify: [read_file] _requires_intersection() — lines ~124-137


BONUS PROBLEM 3: CLI — Unicode encoding crash
───────────────────────────────────────────────────────────────────────────

Przyczyna:
  - Windows default encoding CP1250 nie obsługuje Unicode symboli (✓, ✗)
  - Output kończy się UnicodeEncodeError

Rozwiązanie w main.py:
  
  Przed: print("\n✓ Krzyżówka wygenerowana pomyślnie!")
  Po:   print("\n[OK] Krzyżówka wygenerowana pomyślnie!")
  
  Przed: print("\n✗ Błąd podczas generowania")
  Po:   print("\n[ERROR] Błąd podczas generowania")
